all

This property resets all (apart from unicode-bidi and direction) properties to their initial or inherited state. It accepts the following values: initial, inherit, unset.

initial: changes all the properties of the element or the element's parent their initial value
inherit: changes all the properties of the element or the element's parent to their parent value
unset: changes all the properties of the element or the element's parent to their parent value if they are inheritable or to their own value if not

Documentations say it works in Chrome, Opera, Firefox, IE11 and doesn't work in Safari. I tested it in Safary 10 and it works.

angle~

Angle is a CSS data type. It represents angle values. We are used to using something like "transform: rotate(180deg)". You can not only use "deg", but "grad", "rad" and "turn" as well. This means you can not only use "transform: rotate(180deg)", for example, but transform: rotate(0.5turn) as well. This may be useful for animating elements, when you want to rotate something 4 times, you now can use rotate(4turn).

I tested it in Chrome 54, Opera 41, Firefox 50 and Safari 10.

animation-fill-mode

This property specifies what styles are applied to the element when the animation is not playing. Now this thing is very useful. Imagin if you want you element to fade out. You will create an animation that is running from opacity 1 to 0. After the animation is complete it will jump back to it's initial state. So we actually need the element stay transparent after the animation is completed.

Now let's add animation-fill-mode: forwards; and see how the element keeps the styling it received in the last keyframe. That's when we use the "animation-fill-mode".

Click "Rerun" to see the example again.

It accepts the following values: none, forwards, backwards, both, initial, inherit.

none: (default) Element gets it's initial styling when the animtion is not executing
forwards: animation will apply the values set by the last keyframe executed. Note, that it may not be the state of the "100%" or "to" keyframe. If you set animation-iteration-count to 1.5, the last keyframe will be the 50% keyframe. One more thing that you shall take into account, is that if your animation-direction set to reverse, the last keyframe will be the "0%" or "from" keyframe
backwards: will apply the property values defines in the keyframe that is played at the beginning of the animation
both: the animation will follow the rules of both forwards and backwards
initial: sets animation-fill-mode to it's default value
inherit: element inherits animation-fill-mode value from it's parent

For example you can see in the popular library animate.css, the .animated class uses animation-fill-mode: both

animation-iteration-count

If you have used CSS animations, then you know what "animation-iteration-count" property does. It defines the number of times the animation repeats. But it can also be a non integer value, so animation-iteration-count: 1.5 will play one full animation cycle plus half of the cycle.

backface-visibility

visible: (default) back side of element will be visible, when rotated
hidden: back side of element is not visible
initial: sets the property to it's default (visible) value
inherit: get the property value from it's parent

background-attachment

We sort of forgot this property, as we do not use colorful backgroud images often, but still, it can be useful. If specifies if the background image stays fixed withing the viewport when you scroll the page or scrolls along with the page. It accepts the following values: scroll, fixed, local, initial and inherit.

Scroll: (default) background is fixed to the element and does not scroll with it's contents
Fixed: the background stays fixed when you scroll the page (or element)
Local: scrolls along with the elements contents, if the element is scrollable (elements with "overflow: scroll" for example)

background-blend-mode

This property specifies how element's background images, gradients and colors blend with each other. You can add a background image and background color, then set the blendmode to "lighten". You can also specify several blendmodes, one per background.

The flooowing blending modes can be used:

  • color

  • color-dodgle
  • darken
  • difference
  • exclusion
  • hard-light
  • hue
  • lighten
  • luminosity
  • multiply
  • overlay
  • saturation
  • screen
  • soft-light

background-clip

background-clip defines the area of the element where the background is applied. You can make the background go underneath it's border or fill only the area of it's contents. It accepts the following values: border-box, content-box, padding-box, text, inherit and initial.

border-box: (default) allows background extent to the outer edge of the border
content-box: the background is clipped to the content box of the element
padding-box: the background is clipped to the outer edge of the element's padding, so it's applied to the content, the padding, but not underneath it's border
text: the background is clipped to the foreground text. You can do some cool things using that feature, like this

background-origin

This property defines if the background is applied to only the element's content area, or padding/border areas as well. It accepts this values: border-box, content-box, padding-box, text, inherit and initial.

"How is this different from background-clip?" - you wonder. The difference is that "background-clip" cropes the background, and "background-origin" resizes it.

box-decoration-break

This property specifies how the element's background, padding, border, box-shadow, margin and clip are applied when it's wrapped into multiple lines. If we have our text wrapped intro multiple lines, it would normally increase the element's height, however "box-decoration-break" can apply the styling to each line separately, instead of the whole element. The accepted values are: clone, slice, initial and inherit.

clone: every fragment of the element receives styling, box shadows and padding are applied to each fragment individually
slice: the element is rendered as if the element is not fragmented

Note: Unfortunately you can use it only on inline elements. According to Mozilla documentation you can only use it on Firefox and Chrome inline elements, but I tested it and current browsers show different results. It works on Firefox and Chrome 54, Opera 41 and Safari 10 with the "-webkit" prefix.

calc()

calc() is a function that makes calculations to determine the property value. You can use something like width: calc(100% - 32px). This will make element full width minus 32 pixels. It can be very useful for setting column width and substract the margins. No need for nesting two elements and giving padding to the parent element to achieve the same result.

It accepts an expression with addition, subtraction, multiplication and division operators. Operands can be any syntax lenght values. You can even do something like calc(var(--someValue) / 2 - 2em)

It is still risky to use it on older browsers, although it is accepted by current versions of browsers: http://caniuse.com/#feat=calc

If you use it consider adding fallback properties for old browsers: 

width: 98%; /* fallback for browsers without support for calc() */    
width: calc(100% - 1em);

column-*

"column-count" property specifies the number of columns an element has. column-count: 3 will spread the element text into 3 columns. You can apply it to inline-block, block elements (except table). So this will not work on an element with display: table, but will work on display: table-cell. The fun thing is that it's animatable, so you can change the number of columns with an animation, but unfortunately timing functions will not work on the column-count property.

column-gap: property specifies the size of gap between the columns
column-rule: is a vertical line drawn between the column visually separating them. It's a shorthand for setting this 3 properties (it is very much like border-*)
column-rule-color: specifies the color of the line. The value can be a named color (red, green, transparent), hex, rgb/rgba, hsl/hsla color values
column-rule-style: specifies the style of the line. It accepts the following values: none, hidden, dotted, dashed, solid, double, groove, ridge, inset, outset
column-rule-width: specifies the width of the line and accepts named values (thin, medium, thick), px, em, initial, inherit and unset

Good thing is that this column stuff works on all browsers and comes unprefixed on Chrome 50.

counters

Counters let us give numbers (indexes) to elements. If there are elements on the page that repeat, you can give number to them. For example, you have an article, with several topics, each topic has an H1 heading in the beginning, and you have H2 subheadings. You can automatically number the headings.

There are four properties to work with counters:

counter-reset: resets or creates a counter. the value of this property is the name of the counter that you define
counter-increment: increments the counter. the value is of this property is the name of the counter that you want to increment
content() use :before speudo class and content() to add the index to the elements
counter() function to retreive the value of the counter

So now we know all of this, let's try it in real life: 

body {
  counter-reset: heading; // init the counter for headings
              // note you can give any other name
}
h1 {
  counter-increment: heading; // increments the counter on every H1 tag
  counter-reset: subheading;  // here we init or reset the subheading
                // so that we get 1.1, 1.2, 1.3, then after new heading it will go 2.1, 2.2, 2.3
}
h1:before {
  content: counter(heading) " - "; // using :before selector and counter() function we can add the index to the heading
}
h2 {
  counter-increment: subheading; // increment the subheading counter on every H2 tag
}
h2:before {
  content: counter(heading) "." counter(subheading) " - ";
}

flex

You've probably used it often. It's most often used as flex: 1, but it's actually a shorthand property for flex-grow, flex-shring and flex-basis. The default value for flex is 0 1 auto

flex-basis

flex-basis defines the initial size of a flex element. It accepts width values like px, em and auto. It also accepts intrinsic sizing keywords: fill, max-content, min-content and fit-content. But those work only in Edge, wow!

flex-flow

flex-flow is a CSS shorthand for flex-direction and flex-wrap. It accepts the values of flex-direction (row, row-reverse, column, column-reverse), values of flex-wrap (nowrap, wrap, wrap-reverse). But you can also use it like flex-flow: row nowrap;

font-kerning

this property specifies if the browser shall use kerning information stored in the font or disable it. It accepts the following values:

auto: the browser desides whether to use kerning information stored in font or not
normal: kerning is applied
none: prevent browser from using kerning information

In practice you will probably not use this property, but will use "letter-spacing" for specifing how the space is distributed between letters. Remember, those properties are not the same. Letter-spacing does not affect font kerning, it just increases or descreases distance between letters.

font-smoothing

font-smoothing property controls how anti-aliasing is applied to the font. The problem is that after the standard browser anit-alising your font may look slightly different from what you see in design.

here's the different attitudes Webkit and Firefox have:

-webkit-font-smoothing:

none: turn of anti-alisain, display fonts with sharp edges
antialiased: smooth the font on a pixel level, this make light text look lighter on dark backgrounds
subpixel-antialiased: smooth the font on a subpixel level. this will give the great output on non-retina displays

-moz-osx-font-smoothing:

auto: let browser choose the smoothing method
grayscale: render text with grayscal antialiasing. this gives similar output as "antialised" on webkit, and makes light text look lighter on dark backgrounds.

font-variant

We have an interesting value here: small-caps, which will let us display the text in small capital letters. This can be useful when used together with the CSS ::first-line selector

grid

the grid css layout system lets us create two dimetional grids without problems (by saying problems I mean floats, clears etc). Though it's not safe to use it (as of Dec 2016) it's a good idea to learn how to use it now. Here's a good resource: https://css-tricks.com/snippets/css/complete-guide-grid/

Let's quickly see how this works. Grid consists of the container and the items inside it. Set the following css to the container: display: grid; Then you have to specify how the space is distributed between rows and columns. The values represent column sizes and you can also give names to them. 

.container {
  display: grid;
  grid-template-columns: 40px 50px auto 50px 40px;
  grid-template-rows: 25% 100px auto;
}

Note in the "grid-template-columns" the first and fifth columns have 40px width, the second and fourth columns 50px width, and the column in the middle will get the rest of the space. Let's see how to position item inside the container. We have to actually set the starting and ending points to the element. The item would have the following CSS applied to it: 

.item {
  grid-column-start: 2; // will start at the point where the second column starts
  grid-column-end: 5;   // will end at the point where the second column starts
  grid-row-start: 1;    // will start at the point where the first row starts
  grid-row-end: 3;      // will end at the point where the third row starts
}

Cool, right? You should check whether the browser supports the grid layout. I tell about the @supports media rule here.

hyphens

this propery specifies how the browser hypenates the text.

none: no hypens applied, the text will overflow the container
auto: browser will automatically hypenate the text
manual: manally specify potential break points

Now there are two line break characters you can use:

&shy; - will let the browser know that it's a place in text where the hypenation may be performed if necessary
&hyphen; - will render the hyphen even if the line is not broken

image-rendering

this property specifies how the image is rendered when it's scaled. While it may sound useless, as the browsers automatically apply aliasing, but it has 2 values that can be pretty useful in some cases:

auto: (default) the browser automatically applies aliasing
crisp-egdes: the contrast and edges are preserved, so that they are not smoothed or blured when scaled
pixelated: this value is applied only to the upsized images and makes the image appear pixelated

http://caniuse.com/#search=image-rendering

list-style-image

While most of the time we use list-style-type: none; on the <ul> and then apply background on the <li>, you can still consider applying list-style-image property to the <ul> element to use an image as a marker in the list. 

ul {
  list-style-image: url('sqpurple.gif');
}

list-style-position

This css property specifies where shall the list marker appear. It accepts two values:

inside: markers appear inside the list item content box
outside: markers appear outside the list item content box

object-fit

This is a great property. It specifies how the element is fitted into the box established by it's height and width. For example if you set height and width to an image, you can now choose whether it will shrink or fitted proportionally. Let's take a look at the values it accepts, then see some examples for better understanding.

fill: (default) the contents of the element are size to fill the entire content box (without maintaining the aspect ration)
contain: the contents of the element are fitted inside content box maintaining the aspect ratio
cover: the contents of the element are fitted inside content box maintaining the aspect ratio, but fill the entire space
none: the contents are not resized but cropped using the given height and width
scale-down: automatically chooses between contain and none, selecting the option what would cause the smallest object size

Let's take a look at the examples:

For example without object-fit we had to use this much code in order to get the result of the object-fit: contain; 

.element {
  width: 400px;
  height: 300px;
  position: relative;
}
.element img {
  position: absolute;
  height: auto;
  width: auto;
  max-height: 100%;
  max-width: 100%;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  margin: auto;
}

With object fit it becomes much simpler: 

.element {
  width: 400px;
  height: 300px;
}

img {
  width: inherit;
  height: inherit;
  object-fit: contain;
}

It's very helpful, but the IE and Edge spoil the party. It doesn't work on IE11 and Edge 14 and 15. Opera mini supports it with the prefix "-o-". On the rest of the browsers it works great without any prefixes.

orphans

This property helps you control how the page breaks occur. It accepts a numerical value, that indicated the minimum number of lines of the block container (eg. paragraph) that shall be left at the bottom of the page if the whole container doesn't fit on the page or in the column. Unfortunately it is not currently supported in Firefox.

http://caniuse.com/#search=orphans

order

order property specifies the items within a flex container. the value is a number, which can be negative as well, or "inherit", "initial", "unset". 

.item:nth-child(1) { order: 3 }
.item:nth-child(2) { order: 1 }
.item:nth-child(3) { order: 2 }

This code will display the items in the 2, 3, 1 order.

overflow-wrap

This property specifies if a line break shall occur in long words if they do not fit in the container as one word. accepts the following values:

normal: break words at normal break points
break-word: break normally unbreakable words if there's no space
and the global values: inherit, initial, unset.

page-break-*

There are three CSS properties that allow you adjust page breaks:

page-break-before: controls page breaks before the element
page-break-after: controls page breaks after the element
page-break-inside: controls page breaks inside of the element
This is very convenient when you need to display your pages correctly when printing

They accept the following values:
page-break-before, page-break-after: auto, always, avoid, left, right, recto, verso and global variables(inherit, initial, unset)
page-break-inside: auto, avoid and global variables(inherit, initial, unset)

auto: automatically breaks pages
always: always insert a page break before or after the element (even if there's a lot of space)
avoid: avoid page break before/after/inside of the element if possible
left: insert page break so that the next page is formatted as a left page
right: insert page break so that the next page is formatted as a right page
recto: acts as right if page goes left-to-right, and left if ltr
verso: acts as left if page goes ltr, and right it rtl

There are several things you should know:

  1. "recto" and "verso" values are still experimental
  2. The page-break-* property is going to be replaced by more generic break-* property in the near future
  3. Most of the browsers do support avoid ONLY for page-break-inside (not support for page-break-before and page-break-after)
  4. All browsers except Opera mini, IE and Edge treat "left" and "right" values like "always", so it's a good idea to check if you can use it

percentage~

One thing I shall mention about percentage. We know that percentage is relative to the width of the container. But the properties that require vertical percentage are also calculated based on width of the container. So if we have a container 400px wide 300px tall, padding-top: 10% and padding-left: 10% will both be 40px (not 40px and 30px)!

perspective

The perspective CSS propery gives the element a perspective. it's value is the distance from the viewport to the elements z=0 plane. The value is lenght, that can be represented in px or em, or "none" keyword.

Note: the perspective property affects only 3D transformed elements. The property has to be applied to the parent element (the container) or the element that you want the perspective to be applied on.

pointer-events

This property specifies how elements react to mouse events It accepts the following values:

auto: the element behaves as if the pointer-events are not specified
none: ignores all click, select, drag etc. mouse events

values specific to SVG elements:

visiblePainted: the filled and stroked part of the element are a target. visibily has to be set to visible and stroke value to other than none
visibleFill: only the filled part of the element is a target. visibility has to be set to visible
visibleStroke: only the stroked part of the element is a target. stroke value has to be other than none
visible: the filled and stroked part of the element are a target when the visibility of the element is set to visible
painted: the filled and stroked part of the element are a target. visibility of the element does not affect event processing
fill: only the filled part of the element are a target. visibility of the element does not affect event processing
stroke: only the stroked part of the element are a target. visibility of the element does not affect event processing
all: the whole element is a target. fill, stroke and visibility properties of the element do not affect event processing

global values: inherit, initial, unset

resize

Resize property defines whether the element can be resized by user. You can make an element resizeable vertical, horizontally or both. resize accepts the following values:

none: element is not resizable. This is the default value for most of the element, but textarea has value "both" by default
both: the element is resized both vertically and horizontally
horizontal: user can resize element horizontally
vertical: use can resize element vertical

Note: in order to make an element resizeable, it's visibility has to be set to something other than visible.

shape-outside

This CSS property let's us make the content wrap around an element. Maybe it's a round image and you want text to wrap around that circle. The property shall be applied to the element which you want to wrap the coutent around. It accepts the following values:

Keyword values: none, margin-box, padding-box, border-box, padding-box

Function values: circle(), ellipse(), inset(), polygon()
You can also pass an url to an image around which the content will wrap, and it accepts the global initial, inherit and unset values.

If you choose to use an image, the "shape-image-threshold" property will define the alpha channel threshold to separate the shape from the image.

Firefox and IE still don't have support for this property, and Safari supports it with the "-webkit" prefix. More about this and all of the other properties: https://css-tricks.com/almanac/properties/s/shape-outside/

@supports

This CSS at-rule gives the ability to check if the broser supports the following properties. You can check if it supports a specific value or the property itself: 

@supports (display:grid)
{
  .container {
    display: grid;
  }
}            
    
@supports (image-rendering) {
  img {
    image-rendering: pixelated;
  }
}

You can also check several properties, like this: 

@supports (display:grid) and ((image-rendering:crisp-edges) or (image-rendering:pixelated)) {}

var()

var() function brings the variables functionality from Sass/Less to native CSS.

CSS variables are defined by authors and represent specific values the can be used multiple times in the document. var() function can be used as value of any propery, but cannot be used as a propery or selector. The first value of the function is the name of the custom property; the second is an optional fallback value, which is used if the first argument is invalid.

Let's take a look at the syntax: 

:root {
  --primary-color: #cccccc;
}

body {
  color: var(--primary-color);
}

or me may use the fallback value as well like this: 

body {
  color: var(--primary-color, #cccccc);
}

the same thing looks like this in SASS: 

$primary-color: #ccc;

body {
  color: $primary-color;
}

Unfortunately it's not supported in IE11, Edge 14 and Opera mini. Edge 15 will have partial support.

http://caniuse.com/#search=var

text-transform

While this is a commonly used CSS property, there's one thing I wanted to tell about the "uppercase" transform. The question is which is more correct to use, text typed in CAPS LOCK or to type text in lowercase then transform it to uppercase? The thing is that css transform make it more readable for search engines or screen reader pronunciation. Consider the following Contact us and CONTACT US. A screen would read contact U.S., not us. It's a small nuance, but it's good to know.

transition-timing-function

You know what this propery does if you use CSS transitions or animations. It specifies the speed curve of the transition effect. But here's two little known values:

steps(n,start|end): instead of specifing smooth transiting with such timing functions as ease, this completed the animation in number of steps you define. second parameter is optional and specifies at which point within the interval shall the change occur. The default values for the second parameter is end. You can also use the shorthand functions step-start(n) and step-end(n)
cubic-bezier(n,n,n,n): your own cubic bezier function

Use this tool to create cubic bezier timing functions. You can also use Chrome dev tools to modify the bezier curve.

vh, vw, vmin, vmax

These values are used for sizing things relative to the viewport size. While width is relative to parent container, the vh or vw are always use viewport size as a basis.

vh: 1 vh is equal to 1/100 of viewport height
vw: 1 vw is equal to 1/100 ov viewport width
vmin: it is the least value of vh and vw
vmax: is the largest value of those two

So for a screen that has a viewport 1280x655px, 1vh is equal to 6.55 pixels, 1vw is equal to 12.8pixels, vmin is 6.55 pixels (least value) and vmax is 12.8 pixels (largest value)

white-space

This property defines how the white space inside the element is handled. Let's take a look at the values this property accepts for better understanding.

normal: collapse new lines, collapse spaces and tabs, text wrap
nowrap: collapse new lines, collapse spaces and tabs, text nowrap
pre: preserve new lines, preserve spaces and tabs, text nowrap
pre-wrap: preserve new lines, preserve spaces and tabs, text wrap
pre-line: preserve new lines, collapse spaces and tabs, text wrap

word-break

This property specifies the line breaking rules.

normal: uses standard rules to break words
break-all: lines can be broken between any two letters
keep-all: break are prohibited between letters

word-spacing

This is the equivalent of letter-spacing for words. You can increase or descrease the whitespace between words. the standard value for this property "normal" uses the default size of whitespace (0.25em) any length value, positive or negative, can be used expressed in px, pt, em etc.

will-change

"will-change" property lets authors hind the browsers what changes to expect on an element.

Note: the important thing to remember is that the browser is already trying to optimize the performance of animations and transitions, so you should not use it on many elements, try to use it only when you see that it's necessary. Do not use it for premature optimization.

Let's take a look at the values it accept:

normal: perform standard optimizations
scroll-position: prepare the browser for scroll position changes
content: the contents of the element will change

you can also specify custom properties that will change like this: 

will-change: top, left;

before introduction of this property the following hack to hardware accelerate the animations of an element 

transform: translate3d(0, 0, 0);

writing-mode

This property defines whether the text is layed out horizontally, vertically and the direction.

horizontal-tb: contont flows from left to right, from top to bottom
vertical-rl: content flows from top to bottom, from right to left
vertical-lr: content flows from top to bottom, from left to right

Webkit specific properties

The following properties will be applied only on webkit browsers.

:-webkit-autofill

This CSS pseudo-class mathces input element the is autofilled by the browser. This way you can change the style of an autofilled input in Chrome, because they have yellow background by default.

-webkit-overflow-scrolling

This propertly lets us the the iOS momentum scrolling on elements on touch devices. There are two values:

auto: use normal scrolling, where the scrolling stops at the moment when you remove your finger from screen
touch: use momentum scrolling

-webkit-touch-callout

This property lets us hide the default callout on iOS devices. When you touch and hold the target a menu appears with information and some controls.

none: disables the callout
default: callout is displayed

-webkit-scrollbar

There's a way to hide scrollbar on an element 

#container::-webkit-scrollbar {
  display: none;
}

There's also a way to autohide scrollbards on IE10+ and Edge. 

html {
  -ms-overflow-style: ms-autohiding-scrollbar;
}
